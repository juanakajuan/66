===============================================================================
OVERVIEW                                                           *66-overview*

66 is a Neovim plugin for asking opencode about selected code and making
explicit, localized edits from a visual selection.

66 shells out to the `opencode` CLI from the current working directory.

===============================================================================
SETUP                                                                 *66-setup*

Call setup from your Neovim config: >lua

  require("66").setup()
<

Override defaults by passing an options table: >lua

  require("66").setup({
    model = "openai/gpt-5.5",
    variant = "low",
    agent = "build",
    response_layout = "bottom_split",
  })
<

===============================================================================
COMMANDS                                                           *66-commands*

                                                        *:Ask66*
:Ask66
        Ask opencode about the current visual selection.

                                                        *:Explain66*
:Explain66
        Explain the current visual selection with a preset prompt.

                                                        *:Edit66*
:Edit66
        Ask opencode to edit the current visual selection. Edits are expected to
        stay within the selected lines or immediately adjacent lines.

                                                        *:Search66*
:Search66
        Search the current project and open parsed results in quickfix.

                                                        *:History66*
:History66
        List recent 66-created opencode sessions for the current project and
        open the selected assistant response.

                                                        *:Cancel66*
:Cancel66
        Cancel the newest still-running 66 request in this Neovim instance.

===============================================================================
OPTIONS                                                             *66-options*

All options are passed to `require("66").setup({ ... })`.

                                                        *66-option-model*
model                                                    default: "openai/gpt-5.5"
        opencode model identifier.

                                                        *66-option-variant*
variant                                                  default: "low"
        opencode variant name.

                                                        *66-option-agent*
agent                                                    default: "build"
        opencode agent name.

                                                        *66-option-max_file_lines*
max_file_lines                                           default: 400
        Maximum current-file lines sent with Ask/Explain context.

                                                        *66-option-edit_context_lines*
edit_context_lines                                       default: 120
        Maximum nearby source lines sent with Edit context.

                                                        *66-option-response_layout*
response_layout                                          default: "bottom_split"
        Response View placement. Valid values:
        "bottom_split", "right_split", "float", "tab".

                                                        *66-option-ask_keymap*
ask_keymap                                               default: "<leader>6a"
        Visual-mode Ask About Selection mapping. Use `false` to disable.

                                                        *66-option-explain_keymap*
explain_keymap                                           default: "<leader>6x"
        Visual-mode Explain Selection mapping. Use `false` to disable.

                                                        *66-option-search_keymap*
search_keymap                                            default: "<leader>6s"
        Normal-mode Project Search mapping. Use `false` to disable.

                                                        *66-option-history_keymap*
history_keymap                                           default: "<leader>6h"
        Normal-mode Session History mapping. Use `false` to disable.

                                                        *66-option-edit_keymap*
edit_keymap                                              default: "<leader>6e"
        Visual-mode Edit Selection mapping. Use `false` to disable.

                                                        *66-option-cancel_keymap*
cancel_keymap                                            default: "<leader>6c"
        Normal-mode Cancel Active Request mapping. Use `false` to disable.

===============================================================================
WORKFLOWS                                                         *66-workflows*

Ask About Selection
        Select code in visual mode, run |:Ask66|, write a question in the prompt
        buffer, then `:w` to submit. The answer opens in a scratch Response View.

Explain Selection
        Select code in visual mode and run |:Explain66|. 66 sends a fixed
        explanation prompt and opens the answer in a scratch Response View.

Edit Selection
        Select code in visual mode, run |:Edit66|, write an instruction, then
        `:w` to submit. 66 asks opencode to make a localized edit and applies
        the changed source back into the live buffer.

Project Search
        Run |:Search66|, write the search question, then `:w` to submit. Parsed
        results are opened in the quickfix list. Unparseable output is shown in
        a Response View.

Session History
        Run |:History66| to choose from recent `[66]` opencode sessions for the
        current working directory.

Cancel Active Request
        Run |:Cancel66| to terminate the newest still-running 66 request. Cancel
        does not undo source changes or delete opencode session history.
